# نظام الأرشفة الإلكترونية - آخر تحديث

## وصف النظام

نظام أرشفة إلكتروني مبني باستخدام Flutter Desktop و PHP API و MySQL.

يهدف إلى حفظ الملفات الرئيسية والكتب التابعة داخل هيكل أرشيف منظم على قرص D مع دعم البحث والطباعة والتنبيهات والنسخ الاحتياطي.

---

## هيكل الحفظ

### الملف الرئيسي

يتم إنشاء مجلد خاص بالملف الرئيسي بالشكل التالي:

D:\DocumentArchive\السنة\التصنيف\رقم_الملف_التاريخ_اسم_الملف

وتحفظ الصور داخل:

original

---

### الكتاب التابع

الكتاب التابع لا يعتمد على التصنيف لتحديد مكان الحفظ.

يعتمد فقط على الملف الأصلي المختار.

يتم إنشاء مجلد داخل:

attachments

التابع للملف الأصلي بالشكل التالي:

attachments\رقم_الكتاب_التابع_التاريخ_اسم_الكتاب

مثال:

D:\DocumentArchive\2026\كتب\4641_2026-06-01_اسم_الملف
└── attachments
└── 6158_2026-06-14_اسم_الكتاب

---

## منطق ربط الكتاب التابع

1. اختيار الملف الأصلي من الأرشيف.
2. اعتماد رقم الملف الأصلي والفولدر الخاص به.
3. حفظ الكتاب التابع داخل فولدر الأصل.
4. تسجيل الربط داخل قاعدة البيانات.

---

## التصنيف

* الملف الرئيسي يتطلب تصنيفاً إلزامياً.
* الكتاب التابع لا يتوقف حفظه بسبب التصنيف.
* إذا كان الملف الأصلي يحتوي على تصنيف يتم استخدامه.
* إذا لم يكن التصنيف موجوداً يبقى فارغاً.
* التصنيف لا يؤثر على مكان حفظ الكتاب التابع.

---

## قاعدة البيانات

### جدول الملفات الرئيسية

documents

يحتوي على:

* رقم الملف
* تاريخ الملف
* عنوان الملف
* التصنيف
* الملاحظات
* حالة الإنجاز
* التنبيهات
* مسار الفولدر
* مسارات الصور

---

### جدول الكتب التابعة

document_attachments

يحتوي على:

* رقم الملف الأصلي
* عنوان الملف الأصلي
* رقم الكتاب التابع
* تاريخ الكتاب التابع
* عنوان الكتاب التابع
* التصنيف
* الملاحظات
* التنبيهات
* مسار الفولدر
* مسارات الصور

---

داخل:

---

## ملاحظات مهمة

* لا يتم تعديل أو نقل الملفات القديمة عند تثبيت تحديث جديد.
* جميع الملفات القديمة تبقى متاحة للبحث والاستعراض.
* الكتب التابعة الجديدة يتم حفظها داخل فولدر الملف الأصلي مباشرة.
* في حالة تشابه أرقام الملفات الأصلية يتم اختيار الملف الصحيح من قبل المستخدم.
